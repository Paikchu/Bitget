# Bitget Bot Dashboard — 实现文档

本目录包含从策略机器人升级为可视化交易监控系统的完整实现规划，共分 4 个阶段。

---

## 阶段索引

| 阶段 | 文件 | 主要内容 | 预估工时 |
|------|------|---------|---------|
| Phase 1 | [phase1-database.md](./phase1-database.md) | SQLite 数据库设计 + runner.py 改造 | 1-2 小时 |
| Phase 2 | [phase2-api.md](./phase2-api.md) | FastAPI REST API + WebSocket 实时推送 | 1-2 小时 |
| Phase 3 | [phase3-frontend.md](./phase3-frontend.md) | React Dashboard（K 线图、权益曲线、交易记录、回测面板） | 3-4 小时 |
| Phase 4 | [phase4-deployment.md](./phase4-deployment.md) | Docker 打包 + 服务器部署 + 运维命令 | 30 分钟 |

---

## 最终系统结构

```
┌─────────────────────────────────────────────────────┐
│                     服务器端                          │
│  ┌──────────────┐   ┌─────────────┐   ┌──────────┐  │
│  │  交易机器人   │──▶│  FastAPI    │──▶│  SQLite  │  │
│  │  runner.py   │   │  后端 API   │   │  数据库  │  │
│  └──────────────┘   └──────┬──────┘   └──────────┘  │
│                            │ WebSocket                │
└────────────────────────────┼────────────────────────┘
                             │
                    ┌────────▼────────┐
                    │  React 前端     │
                    │  K线图 + 权益   │
                    │  + 交易记录     │
                    │  + 回测面板     │
                    └─────────────────┘
```

## 推荐实施顺序

按 Phase 1 → 2 → 3 → 4 顺序执行，每个阶段完成后均可独立验证，不影响现有回测和交易功能。
